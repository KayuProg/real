温湿度センサ

周囲の温度と湿度の取得には温湿度センサを用いる． ＤＨＴ－１１を使うことをはじめは考えたが次のサイトから https://zenn.dev/suzuky/articles/25dab74334e6ff 個体差があり，10%-15%もの誤差があること，湿度100%まで計測できないことから，DHT-22という上位モデル？を使用することにした． GND/DATA/VDDの三本からなる．データはシングル・バス・データフォーマットという形式である． これはライブラリで解決できる． 　DATAピンはプルアップして使用する．抵抗はデータシートで推奨されている5kΩを使用した． https://iot.keicode.com/arduino/temperature-dht11.php 　も参照とした． Sketch uses 5058 bytes (16%) of program storage space. Maximum is 30720 bytes. Global variables use 257 bytes (12%) of dynamic memory, leaving 1791 bytes for local variables. Maximum is 2048 bytes. avrdude: ser_open(): can't open device "\\.\COM20": Access is denied. というエラーについて． このエラーは原因がよくわからない場合シリアルモニタによるものだと考えられる． Arudinoを使っていて書き込もうとした際に以前書き込んでいたプログラムがシリアルモニタに通信するものであると， そのプログラムが動作している限りそのUSBポートは以前のプログラムに占用されていることになる． したがって，書き込みをする際にはシリアルポートを閉じることでマイコンからのUSBシリアル通信をなくしポートを開放することができる． 以上によりこのエラーは解消される． 温度と湿度の関係について室温が25-28度の時湿度は50%-60% 室温が18-25度の時は湿度は40-50%くらいが快適とされている． https://myhome.nifty.com/column/kaji/230626360888/ これに従ってプログラムを組んでいく．

プログラム

プログラムを組んでいく．今回，計測した室温から適切な湿度になるようなフィードバック制御を行う． まず，今回使用する加湿器の仕組みは圧電セラミックスを用いた超音波ミスト生成であった． 圧電振動子が超音波を発生させることで振動し，小さな穴から水蒸気が噴出されるという仕組みである． https://www.tdk.com/ja/tech-mag/hatena/044 ここで，この加湿器についている圧電セラミックス（素子）について調べる． おそらく同じものを発見した． https://www.amazon.co.jp/gp/product/B099NSXTND?ie=UTF8&psc=1&linkCode=sl1&tag=motochi1206-22&linkId=c2a96756a47287df0081a6f5d3e37e38&language=ja_JP&ref_=as_li_ss_tl ミストメーカ この加湿器でミストを発生させている際の電圧を測定すると3Vくらいであった． なので，ミストを発生させたいときは振動子に3V加えればよい． ある一定の湿度を保つために，累積誤差を考えてプログラムを組んだ． 制御するのは振動子に加える電圧で発生するミストの量と関係している．（おおよそ比例関係） スライドスイッチを使ってミスト発生モジュールの制御をもとのやつか，このマイコンかを切り替えるようにする．
