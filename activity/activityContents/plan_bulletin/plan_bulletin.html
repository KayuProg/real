契機

普段自分のタスク管理は，Google CalenderとiOSのリマインダーを使っています．

その日の大きなイベント（テストだったり，レポート提出だったり）はGoogle Calender， 細かいタスク（その日の課題だったり，買い出し）はリマインダーという使い分けです．

二つアプリを使っているため，予定確認の際にアプリを行き来しなくてはいけないことがあり面倒です．

なので当日の予定を一画面に表示してくれるものを作ります．

Raspberry pi3 model B+を使います．

iOSリマインダー

まず，iOSのリマインダーからデータを取得しようとした．

方法としてpythonのreminder moduleを見つけた．

pip installしたら次のエラーが出た．

ERROR: Could not find a version that satisfies the requirement reminders

これはhttps://qiita.com/noppe78/items/cc5327ecdf33103b377bによるとPythonバージョンとの非互換性によるエラーらしい

解決方法としてPyenvによるPython version切り替えがあるそうです．

方法としては，iPhone上でPython プログラムを定期的に実行してクラウドにリマインダーのデータをアップロードすることでラズパイに取得させる方法を考えました．

面倒ですね．

この方法はなしで

結局，Google Tasksというリマインダーアプリに移行することにしました．

（Google Calender使っているから同じGoogleのアプリならコード書きやすいのではないかという考え）

なぜなら，Google TasksにはAPIがあるからです．

さすが天下のGoogle様．

以降に手間はかかりますが，ラズパイからデータが取得できるようになりました．

Python 3.12 install

Google TasksのAPIを使うためにはPython 3.10.7以上が必要みたいです．

せっかくだしもう少し新しいやつをinstallします．

https://blog.goo.ne.jp/espiya/e/6b7f4d66b7253b4eb31cce32e0609a30にしたがってやります．

ただし，インストールした場所は/usr/bin/pythonです．（注意！） さっきのできなかったからこっちのサイト https://qiita.com/guijiu/questions/7a3fae535edef4ea2e49

たまにPython-3.12.3.tar.xz: Permission denied Cannot write to ‘Python-3.12.3.tar.xz’ (Permission denied). というエラーが出るので実行するコマンドにsudoつけるのがキモ．

https://qiita.com/hoshi621/items/c8b82ab42564256df884 ここでpyenvはinstall下． installできないときはメモリが足りない場合があるので， sudo fallocate -l 2G /swapfile sudo chmod 600 /swapfile sudo mkswap /swapfile sudo swapon /swapfile でスワップを増やしたらできます．

pyenv not commandみたいな感じになったら$ source ~/.bash_profile 読込なおせば行ける

仮想環境を構築してversionを変更する方法．さらに，仮想環境状態でのみpythonのversionを変更する．

kayu@raspberrypi:~ $ cd /home/kayu/Desktop/test/plan/　#仮想環境を作りたいフォルダに移動 kayu@raspberrypi:~/Desktop/test/plan $ python -V Python 3.9.2 kayu@raspberrypi:~/Desktop/test/plan $ pyenv versions　#現在のこのpython version確認 system * 3.9.2 (set by /home/kayu/Desktop/test/plan/.python-version) 3.12.3 kayu@raspberrypi:~/Desktop/test/plan $ pyenv local 3.12.3　#一度この仮想環境にしたいフォルダを使いたいpython versionに変更 kayu@raspberrypi:~/Desktop/test/plan $ python -V Python 3.12.3 kayu@raspberrypi:~/Desktop/test/plan $ python -m venv .venv　#このpython versionが3.12.3の状態で仮想環境構築（こうするとpytohn 3.12.3の仮想環境ができる） kayu@raspberrypi:~/Desktop/test/plan $ . .venv/bin/activate　　#仮想環境有効化 (.venv) kayu@raspberrypi:~/Desktop/test/plan $ python -V　#仮想環境状態でpython versionが3.12.3になっているか確認 Python 3.12.3 (.venv) kayu@raspberrypi:~/Desktop/test/plan $ deactivate　#仮想環境状態終了 kayu@raspberrypi:~/Desktop/test/plan $ pyenv local 3.9.2　#仮想環境をオフにした状態でpythonのversionを戻す． kayu@raspberrypi:~/Desktop/test/plan $ pyenv versions　#python versionが3.12.3ではなく3.9.2であることを確認． system * 3.9.2 (set by /home/kayu/Desktop/test/plan/.python-version) 3.12.3

https://developers.google.com/workspace/guides/create-project?hl=ja ここでGoogle Cloudでprojectを作成する． https://www.tdk.com/ja/tech-mag/knowledge/147 OAuthとは plan_show_test は Google の審査プロセスを完了していません。このアプリは現在テスト中で、デベロッパーに承認されたテスターのみがアクセスできます。アクセス権があると思われる場合は、デベロッパーにお問い合わせください。 というエラーが出た．なんだと．．． Google Cloudの確認ステータスで「検証は不要です」となるようにスコープ？権限を調整（全てオフにした）したら行けた． やった！取得できた！ https://messefor.hatenablog.com/entry/2020/10/08/080414 ここがわかりやすかった．APIキー，クライアントIDなどの解説

取得した内容はこんな感じで返ってくる． [{'kind': 'tasks#task', 'id': 'ZlBHdmtHbUM4R2l0X29sSA', 'etag': '"LTU5MDgzMzAzNQ"', 'title': 'タスク1', 'updated': '2024-10-30T09:36:27.000Z', 'selfLink': 'https://www.googleapis.com/tasks/v1/lists/MTQzMjUyMTY0MTk3NDc0NDg1ODU6MDow/tasks/ZlBHdmtHbUM4R2l0X29sSA', 'position': '00000000000000000000', 'status': 'needsAction', 'due': '2024-10-31T00:00:00.000Z', 'links': [], 'webViewLink': 'https://tasks.google.com/task/fPGvkGmC8Git_olH?sa=6'}] dueがこのタスクの期日になるのだ，RFC3339 timestampというのが使われていた． https://developers.google.com/resources/api-libraries/documentation/tasks/v1/python/latest/tasks_v1.tasks.html#list に書いてあった．

それに日付だけしかAPIで引っ張れないらしい．う～ん． " "due": "A String", # Due date of the task (as a RFC 3339 timestamp). Optional. The due date only records date information; the time portion of the timestamp is discarded when setting the due date. It isn't possible to read or write the time that a task is due via the API." 'due': '2024-10-31T00:00:00.000Z', こんな感じで時刻（taskの期限）がかえってきます． https://www.jisakeisan.com/?y=2024&m=10&d=30&hh=18&mm=47&t1=jst&t2=utc このサイトを見ながらUTC 協定世界時とJST 日本標準時の変換を行おうとしたのですが，taskの日にちだけ記録されているようです．

つまり，taskの期日を1/2 7:00にしたとしてUTCだと1/1 22:00だから1/1と記録されるとかになるとかではない．ということでした． なので，このまま日付だけ使っていきます．

同様に似たGoogle calender用のページでGoogle Calenderからも情報をとれるようにしました．

ディスプレイに表示させる方法として考えたのは2通り

・ラズパイ上でWebサーバを構築してWeb上で表示させる．

・デスクトップアプリを作ってその上に表示させる．

Webサーバ構築のほうは時間がかかる上にWebを定期的に自動更新させる必要が出てくる．

HTMLをpythonで編集していく形になると思うが内容が随時更新されるようにできるだろうか．

うーん．デスクトップアプリ作ったほうが早そうだし，簡単そう．

pythonでデスクトップアプリ作ります．

ただ文字列を表示させて定期的に更新するだけだからpythonでデスクトップアプリ作っても問題ないだろう．

リアルタイム性が重要でもないしな．

ラズパイデスクトップからpythonスクリプト実行

デスクトップにシェルスクリプトを作成する． 内容は次のように編集 # 実行権限を付与。これでダブルクリックした際に「端末で実行」が出てくる cd /home/kayu/Desktop/plan/#動作させたいフォルダまで移動 chmod +x ./test/a.py#実行権限の付与 python test/a.py#pythonを実行． 注意しなくてはいけないのが，実行したいpythonスクリプト内で相対パスなどをしている場合，次の内容をpythonスクリプト内に書く必要がある．

import os.path os.chdir('/home/kayu/Desktop/plan/')#パスは実行したいスクリプトのあるフォルダ これめっちゃ大事！このせいで少し沼りかけました．

UIについて

iPadに文字を表示するにあたってデスクトップアプリを作成することにした．

そこで，以前使ったことのあるTkinterを使おうとしたが，UIの作成にかなり苦労したことを思い出した．

そこでpythonで作るデスクトップアプリについて調べていたら次のようなものを見つけた． https://qiita.com/ForestMountain1234/items/bc709c3599ee86e2a0dd Fletです！ https://qiita.com/NasuPanda/items/48849d7f925784d6b6a0 軽く記事呼んだだけでもこれはやばい．すごいやつだ．

すごい簡単にデスクトップアプリへのビルド，Webアプリとしてデプロイもできるらしい．

ほんとすごい． Webと同じ感覚でflexbox的に配置できるのも素晴らしい． これを使おう． ドキュメントはこちらhttps://flet.dev/docs/getting-started/ ドキュメントでは仮想環境でのapp製作を進められているのでpyenvの仮想環境で進めていきます． 注意！WSL or Linux環境でのinstallはいろいろinstall必要みたいだからドキュメントちゃんと読む！ で，Audio使ったflet appのbuildには--include-packages flet_audioというコマンドが必要らしい．

flet build apk --include-packages flet_audioこんな感じらしい． videoを使う場合も追加でinstall必要みたい．（ドキュメント自分でよんでね）

Fletでデスクトップアプリを作る

flet runで以下のエラーが出た．

error while loading shared libraries: libmpv.so.1: cannot open shared object file: No such file or directory 共有ライブラリが見つからないらしい https://qiita.com/kawamo55/items/4d973ede723f2987ef7a 結局このサイトで行けそう． またエラー出た．（いつも通り） Unable to init server: Could not connect: Connection refused これは単に自分の場合はラズパイのデスクトップを開いてないことによるものだった． ずっとvscodeでssh接続で捜査していた． flet runをしたい場合はvnc viewerなどでラズパイのデスクトップを直接表示させて実行する必要があった． しかし，またエラー atk_socket_embed: assertion 'plug_id != NULL' failed うーん．しらべてもよくわからん． そこで一度ラズパイを再起動した．
